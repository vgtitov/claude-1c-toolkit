# Чек-листы качества: готовность, приёмка, архитектура

Контекст: этот reference — набор чек-листов для контроля качества требований, задач и архитектурных решений и для
организации приёмки. Дополняет чек-лист «хорошего ЧТЗ» из `analysis-workflow.md`. Общеизвестная методика (INVEST,
DoR/DoD, этапы внедрения), org-agnostic.

## Definition of Ready (готовность к разработке)
Задача готова к разработке, если:
- соблюдён **INVEST** (см. `requirements-and-chtz.md`);
- в описании есть: роль-заказчик (потребитель функционала); описание доработки (что реализовать); измеримая выгода;
  масштабируемость (на какие базы/контуры распространяется и как);
- приложен сценарий тестирования в формате **AS-IS / TO-BE** (по шагам, чтобы разработчик прошёл и проверил).

## Definition of Done (готовность результата)
Результат готов, если:
- критерии приёмки выполнены и проверены;
- тесты пройдены;
- документация обновлена.

## Чек-лист качества требования
- [ ] Однозначность (нет двойных толкований).
- [ ] Проверяемость (есть измеримый критерий приёмки).
- [ ] Нет деталей реализации (иначе — в тех-проект).
- [ ] Трассируемость (есть осмысленный идентификатор).
- [ ] Полнота краевых и ошибочных кейсов.

## Чек-лист качества архитектурного решения
- [ ] Соответствие корпоративной/целевой архитектуре.
- [ ] Перечислены затрагиваемые и изменяемые области (оба слоя контура: конфигурация + расширение).
- [ ] Оценены риски.
- [ ] Зафиксированы рассмотренные и отвергнутые варианты (ADR) — см. `architecture-design.md`.

## Этапы внедрения и приёмка (ERP-канон)
Типовая последовательность этапов: **Подготовка → Проектирование → Реализация → подготовка к ОПЭ/ОЭ → ОПЭ/ОЭ →
переход к ПЭ → ПЭ** (ОЭ — опытная эксплуатация, ОПЭ — опытно-промышленная, ПЭ — промышленная).
- **Реестр функциональных разрывов** — артефакт этапа проектирования (разрывы между типовой и потребностью).
- **Программа и методика испытаний** — основа приёмки (по ГОСТ 19.301-79 в проектах, где применима ГОСТ-номенклатура).
- Состав приёмки: состав и объём испытаний → приёмочное тестирование по критериям приёмки → регистрация и устранение
  дефектов → ввод в эксплуатацию.

## Как этим пользоваться
Прогонять DoR перед передачей задачи в разработку, чек-лист требования — при подготовке ЧТЗ, чек-лист архитектуры —
когда подключается архитектор. Несоответствие любому пункту = открытый вопрос/блокер, а не «домыслить и идти дальше».
