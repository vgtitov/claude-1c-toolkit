# Стандарты кода 1С (BSL) — distilled для ревью/написания

Источник: официальные «Стандарты разработки» 1С (`1C-Company/v8-code-style`, встроены в 1C:EDT) + диагностики
**BSL Language Server** (`1c-syntax`, headless/MCP-мост). Два движка реализуют одни стандарты ИТС, но КЛЮЧИ
диагностик разные: ниже даны оба (`v8-code-style` / `BSL LS`). Стандарты версионно-нейтральны (это стиль/архитектура,
не API). Учитывай режим совместимости СВОЕЙ конфигурации.

Полные списки (не дублируем, читать по необходимости):
- BSL-проверки (~88): https://raw.githubusercontent.com/1C-Company/v8-code-style/master/docs/checks/bsl.md
- Запросы (8): https://raw.githubusercontent.com/1C-Company/v8-code-style/master/docs/checks/ql.md
- Типизация/докомментарии: https://raw.githubusercontent.com/1C-Company/v8-code-style/master/docs/checks/code_typification.md
- Формы / метаданные / права: form.md · md.md · right.md (там же)
- BSL LS диагностики (180): https://1c-syntax.github.io/bsl-language-server/en/diagnostics/

## Порядок применения
После написания/правки BSL — прогнать **BSL LS** (headless/MCP-мост). Трактовать: `Error` — блокер;
`Vulnerability`/`Security Hotspot` — обязательны к ревью; `Code smell` — рекомендация. EDT-проверки
(v8-code-style) недоступны headless → использовать как чек-лист ревью. До 3 циклов автоисправления, затем —
к человеку. Изменения метаданных/форм/СКД — только в EDT (см. SKILL.md, запреты AI).

## Топ-25 правил (формулировка — v8-code-style / BSL LS)
1. Запрос внутри цикла запрещён — `query-in-loop` / `CreateQueryInCycle`.
2. `НачатьТранзакцию()` строго перед `Попытка` — `begin-transaction` / `BeginTransactionBeforeTryCatch`.
3. `Зафиксировать/ОтменитьТранзакцию` только внутри `Попытка`, парно — `commit/rollback-transaction` / `CommitTransactionOutsideTryCatch`, `PairingBrokenTransaction`.
4. Пустой блок `Исключение` запрещён (минимум логирование) — `empty-except-statement`.
5. `Заблокировать()` только внутри `Попытка` — `lock-out-of-try`.
6. Стандартная структура модуля: области верхнего уровня в правильном порядке — `module-structure-top-region` (стандарт 455) / `CodeOutOfRegion`.
7. Все методы — внутри областей; код инициализации — в своей области — `module-structure-method-in-regions`.
8. Имя общего модуля = постфикс по флагам компиляции (Клиент/Сервер/ВызовСервера/ПовтИсп/ПолныеПрава) — `common-module-name-*` / `CommonModuleName*`.
9. Объекты/методы/переменные расширения — с префиксом своего расширения — `extension-*-prefix`. См. conventions-template.md.
10. В расширении предпочесть `&ИзменениеИКонтроль` вместо `&Вместо` — `change-and-validate-instead-of-around` (если доступно в твоём режиме совместимости, 8.3.18+).
11. Перед `Выполнить()`/`Вычислить()` на сервере — `УстановитьБезопасныйРежим` — `server-execution-safe-mode` / `ExecuteExternalCode`.
12. Никаких хардкод-секретов/паролей — `unsafe-password-ib-storage` / `UsingHardcodeSecretInformation`.
13. Экспортные процедуры/функции — с документирующим комментарием (Параметры, Возвращаемое значение) — `doc-comment-*`.
14. Функция возвращает значение по всем веткам — `function-return-value-type` / `FunctionShouldHaveReturn`.
15. Неиспользуемые переменные/методы/параметры — удалять — `module-unused-*` / `UnusedLocalVariable`, `UnusedLocalMethod`.
16. Лишние экспортные методы (не вызываются извне) — не делать экспортными — `redundant-export-method`.
17. Необязательные параметры — только после обязательных — `method-optional-parameter-before-required`.
18. Слишком много параметров — рефакторить в структуру — `method-too-many-params`.
19. Точечное чтение реквизита из БД (`Ссылка.Реквизит`) массово/в цикле — заменить запросом — `reading-attribute-from-database`.
20. `Перейти` (Goto) запрещён — `use-goto-operator`.
21. Один обработчик — на одно событие/команду формы — `form-items-single-event-handler`.
22. Устаревшие методы/конструкции (`Новый Цвет/Шрифт` → стилевые) — `use-non-recommended-method`, `new-color`, `new-font`.
23. Устаревшие процедуры — только в области `УстаревшиеПроцедурыИФункции` — `deprecated-procedure-outside-deprecated-region`.
24. Сложность/размер метода в пределах (декомпозировать) — только BSL LS: `CognitiveComplexity`, `CyclomaticComplexity`, `MethodSize`, `NestedStatements`.
25. Дорогой публичный API — кэшировать (ПовтИсп); магические числа/даты/дубли литералов выносить — `public-method-caching`, BSL LS `MagicNumber`, `MagicDate`, `DuplicateStringLiteral`.

## Порядок областей модуля (стандарт 455) — фиксировать явно
- **Общий модуль:** `ПрограммныйИнтерфейс` → `СлужебныйПрограммныйИнтерфейс` → `СлужебныеПроцедурыИФункции`
  (опц. `ИнициализацияМодуля` в конце).
- **Модуль формы:** `ОбработчикиСобытийФормы` → `ОбработчикиСобытийЭлементовШапкиФормы` →
  `ОбработчикиСобытийЭлементовТаблицФормы` → `ОбработчикиКомандФормы` → `СлужебныеПроцедурыИФункции`.
- **Модуль объекта:** `ОбработчикиСобытий` → `СлужебныеПроцедурыИФункции`.
Директивы компиляции (`&НаКлиенте`/`&НаСервере`/`&НаСервереБезКонтекста`) — по назначению; на форме обязательны.

## Чего НЕ копировать из community-правил
Async `Асинх`/`Ждать` и «вайбкод»-паттерны форм из некоторых открытых наборов рассчитаны на 8.3.18+ и свободную
правку форм. Если твой режим совместимости ниже — async недоступен; плюс действует IDE-гейт (формы/СКД меняются
только в IDE). Брать осторожно, помечать `[проверить]`.
