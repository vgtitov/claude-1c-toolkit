# ai_debug — отладочное расширение слоя данных (L3)

Минимальное расширение 1С (совместимость **8.3.20+**, префикс `aidbg_`): HTTP-сервис
`/hs/aidbg` для MCP-сервера `onec-data` (`mcp/onec_data_mcp.py`). Даёт AI-агенту
**read-only** доступ к данным живой ИБ ОТ ИМЕНИ аутентифицированного пользователя:
произвольный запрос, СКД-отчёт, настройки, вызов метода из белого списка.
Архитектура: `docs/data-access-architecture.md` (лестница L0–L4).

## Контракт HTTP (все ответы JSON)

| Маршрут | Метод | Тело | Ответ |
|---|---|---|---|
| `/hs/aidbg/query` | POST | `{text, params, mode, top}` | `{rows, fetched, shown}` |
| `/hs/aidbg/report` | POST | `{report, variant, params, filters, mode, top}` | `{rows, fetched, shown, variant, filters_applied}` |
| `/hs/aidbg/settings` | POST | `{what: constants\|functional_options\|user_settings\|access, names, params, target_user, mode}` | значения |
| `/hs/aidbg/call` | POST | `{method, args, mode}` | `{method, result}` |
| `/hs/aidbg/health` | GET | — | `{version, session_user, privileged_allowed, methods_whitelist, max_rows}` |

Типизация значений в `params`/`args`: примитивы как есть; дата `{"date": "2026-07-01T00:00:00"}`;
ссылка `{"ref": "Справочник.Номенклатура", "uuid": "…"}`; перечисление
`{"enum": "Перечисление.СтавкиНДС", "value": "НДС20"}`; массив — для `В СПИСКЕ`.
Ссылки в ответах: `{_ref, uuid|value, presentation}`.

## Режимы (`mode`)

- `user` (по умолчанию) — сеанс пользователя из Basic-auth, **RLS честный**. Для отладки
  «что видит пользователь», RLS-матриц, воспроизведения инцидентов.
- `privileged` — `УстановитьПривилегированныйРежим(Истина)` на время выполнения: RLS
  обходится, «эталон всё». Работает ТОЛЬКО если на базе
  `aidbg_Настройки.ПривилегированныйРежимРазрешен()` = Истина (тест-контур).

## Установка (тест-база; прод — только за белым списком на веб-сервере)

1. Загрузить `src/` как расширение (Конфигуратор: Расширения конфигурации → загрузить
   из файлов; или собрать .cfe). Назначение: Дополнение, безопасный режим по задаче.
2. **Настроить `aidbg_Настройки`** — единственный правимый файл:
   `ПривилегированныйРежимРазрешен()` (Истина только на тест-копии),
   `РазрешенныеМетоды()` (whitelist читающих методов для `/call`), `МаксимумСтрок()`.
3. Опубликовать ИБ с включёнными HTTP-сервисами (галка у `aidbg_API`).
   ⚠️ Регистр имени базы в URL публикации должен совпадать точно (иначе редирект
   превращает POST в GET).
4. Проверка: `GET /hs/aidbg/health` под двумя разными пользователями — в `session_user`
   должен вернуться каждый из них.
5. Для RLS-отладки исследуемым пользователям нужен доступ по паролю (Basic-auth) и права
   на использование HTTP-сервисов.

## Гарантии безопасности (проверять при любом ревью)

1. **Записи данных нет by design** — ни одного метода записи в модулях.
2. Свойство `Privileged` у общих модулей = Ложь; привилегированный режим только явный
   и только за гейтом `aidbg_Настройки` (одна точка решения — `ПроверитьРежим`).
3. `/call` — только методы из whitelist + валидация формата имени (защита `Вычислить`).
4. `/query` принимает только тексты, начинающиеся с ВЫБРАТЬ/SELECT (язык запросов 1С
   и так не пишет данные — это защита в глубину).
5. Серверный потолок строк (`МаксимумСтрок`) режет любой ответ.
6. Аудит каждого вызова в журнал регистрации: события `ai_debug.query|report|settings|call`.
7. Маскирование ПДн выполняет клиент (`onec_data_mcp.py`), включено по умолчанию.

## Хвост внедрения

Синтакс-контроль платформой и round-trip на живой базе выполняются на машине с 1С
(см. `docs/data-access-implementation-plan.md` §4.4, §6) — расширение написано по
документированному API платформы, но до загрузки в тест-базу считается непроверенным.
