# Топ-методы БСП 3.1.9 — писать через них, а не «свой велосипед»

Сигнатуры сняты из реального исходника БСП **3.1.9.144** (`ssl_3_1_9/src/cf/CommonModules/*`), точны для 3.1.9.
Перед использованием в незнакомом месте — сверить через MCP кода (`search_1c "<имя>" scope=baseline`) или Grep по
`ssl_3_1_9`. Модуль указан над группой; КС = КлиентСервер.

## Реквизиты объектов — ГЛАВНОЕ (не читать в цикле через точку)
Модуль `ОбщегоНазначения` (сервер). `Реквизиты` — строка через запятую ИЛИ массив; поддерживает алиасы
`"Контрагент КАК Клиент"`. `ВыбратьРазрешенные = Истина` — учитывать RLS.
- `ЗначениеРеквизитаОбъекта(Ссылка, ИмяРеквизита, ВыбратьРазрешенные=Ложь, КодЯзыка=Неопределено)` → значение.
- `ЗначенияРеквизитовОбъекта(Ссылка, Реквизиты, …)` → **Структура** {имя→значение} за один запрос.
- ⭐ `ЗначенияРеквизитовОбъектов(Ссылки, Реквизиты, …)` → **Соответствие** Ссылка→Структура — ПАКЕТНО, один
  запрос на массив. Это замена `Стр.Номенклатура.Артикул` в цикле.
- `ЗначениеРеквизитаОбъектов(МассивСсылок, ИмяРеквизита, …)` → Соответствие Ссылка→значение (один реквизит, пачкой).
- `УстановитьЗначениеРеквизита(Объект, Имя, Значение, …)` / `УстановитьЗначенияРеквизитов(Объект, Значения)`.
- `ОбщегоНазначенияКлиентСервер.ЕстьРеквизитИлиСвойствоОбъекта(Объект, ИмяРеквизита)` → Булево.

Пример (один запрос вместо запроса-на-строку):
```bsl
МассивНом = Таблица.ВыгрузитьКолонку("Номенклатура");
Данные = ОбщегоНазначения.ЗначенияРеквизитовОбъектов(МассивНом, "Артикул, ЕдиницаИзмерения");
Для Каждого Стр Из Таблица Цикл
    Артикул = Данные[Стр.Номенклатура].Артикул;   // из памяти, без обращения к БД
КонецЦикла;
```

## Массивы и коллекции
Модуль `ОбщегоНазначенияКлиентСервер` (если не указано иное).
- `СвернутьМассив(Массив)` → без дубликатов. `ДополнитьМассив(Приемник, Источник, ТолькоУникальные=Ложь)`.
- `РазностьМассивов(Массив, Вычитание)`, `СократитьМассив(Массив, Вычитание)`.
- `УдалитьЗначениеИзМассива(М, Знач)`, `УдалитьВсеВхожденияЗначенияИзМассива(М, Знач)`, `…ТипаИзМассива(М, Тип)`.
- `ЗначениеВМассиве(Значение)` → массив из 1 элемента (удобно для параметров запроса).
- `СвойствоСтруктуры(Структура, Ключ, ЗначениеПоУмолчанию=Неопределено)` → безопасное чтение ключа.
- `ДополнитьСтруктуру/ДополнитьСоответствие(Приемник, Источник, Заменять)`, `КлючиСтруктурыВСтроку(Стр, Разд)`.
- `СкопироватьРекурсивно(Источник)` (КС) / `ОбщегоНазначения.СкопироватьРекурсивно(Источник, ФиксироватьДанные)`
  — глубокая копия. Плоские: `СкопироватьСтруктуру/Соответствие/Массив/СписокЗначений(Источник)`.
- `ОбщегоНазначения`: `ТаблицаЗначенийВМассив(ТЗ)`, `ВыгрузитьКолонку(КоллекцияСтрок, ИмяКолонки, ТолькоУникальные=Ложь)`,
  `СтрокаТаблицыЗначенийВСтруктуру(Строка)`.

## Строки
Модуль `СтроковыеФункцииКлиентСервер`.
- ⭐ `ПодставитьПараметрыВСтроку(Шаблон, П1…П10)` → подстановка `%1..%10`. Базовый способ формирования сообщений:
  `ПодставитьПараметрыВСтроку(НСтр("ru='Заказ %1 от %2'"), Номер, Дата)`.
- `ПодставитьПараметрыВСтрокуИзМассива(Шаблон, Параметры)`, `ВставитьПараметрыВСтроку(Шаблон, Параметры)` (по `%Ключ`).
- `РазложитьСтрокуВМассивСлов(Значение, Разделители=Неопределено)`, `ПараметрыИзСтроки(Стр, Разд=";")` → Структура.
- `ДополнитьСтроку(Знач, Длина, Символ="0", Режим="Слева")`, `СформироватьСтрокуСимволов(Символ, Длина)`.
- `ТолькоЦифрыВСтроке/ТолькоЛатиницаВСтроке/ТолькоКириллицаВСтроке(...)` → Булево.
- `ФормаМножественногоЧисла(Для1, Для2, Для5, Значение)`, `ИзвлечьТекстИзHTML(Текст)`.
- Регистр строк — платформенные `ВРег()/НРег()/ТРег()`, отдельной обёртки БСП нет.

## Данные / менеджеры / права
Модуль `ОбщегоНазначения` (сервер).
- `МенеджерОбъектаПоСсылке(Ссылка)`, `МенеджерОбъектаПоПолномуИмени("Документ.РеализацияТоваровУслуг")`.
- `ОбщийМодуль(Имя)` → вызвать общий модуль по имени. `ИмяТаблицыПоСсылке(Ссылка)` (для запросов).
- `ЭтоСсылка/СсылкаСуществует/ЗначениеСсылочногоТипа(...)` → Булево; `ОбъектЯвляетсяГруппой(Объект)`.
- `ЕстьСсылкиНаОбъект(СсылкаИлиМассив, ИскатьСредиСлужебных=Ложь)` — контроль перед удалением.
- `ПроверитьПроведенностьДокументов(Док)`, `ПровестиДокументы(Док)` → массив ошибок.
- `СформироватьСтрокуДляПоискаВЗапросе(Строка)` — экранирование для `ПОДОБНО`.
- `ПредопределенныйЭлемент("Справочник.Валюты.Рубль")` (есть и в …Клиент/…КлиентСервер) — вместо
  `Справочники.Валюты.Рубль`; нужно для расширений/мультиконфигурации.
- `ДанныеСовпадают(Д1, Д2)` — глубокое сравнение. `ФиксированныеДанные(Данные, ВызыватьИсключение=Истина)` —
  сделать структуру/массив/соответствие неизменяемыми (для безопасной отдачи из ПовтИсп).
- Права: параметр `ВыбратьРазрешенные=Истина` в методах реквизитов (фильтрует по RLS) + подсистема `Пользователи`
  (`Пользователи.РолиДоступны(...)`, `Пользователи.ЭтоПолноправныйПользователь()` — точные имена сверять).
- Привилегированный режим: `УстановитьПривилегированныйРежим(Истина/Ложь)` (платформенный) — для служебных операций.

## Прочее частое
- `ОбщегоНазначения.ТекущаяДатаПользователя()`, `РабочаяДатаПользователя(...)`.
- `…КлиентСервер.РаспределитьСуммуПропорциональноКоэффициентам(Сумма, Коэффициенты, Точность=2)` — разнос без расхождений.
- `…КлиентСервер`: `ПолучитьПолноеИмяФайла/РазложитьПолноеИмяФайла/ДобавитьКонечныйРазделительПути/`
  `ПолучитьРасширениеИмениФайла/ЗаменитьНедопустимыеСимволыВИмениФайла` — кроссплатформенные пути (важно mac+win).
- `…КлиентСервер.СтруктураURI(URI)`, `…Клиент.ОповеститьОбИзмененииОбъекта(Ссылка)`.
- Сериализация: `ОбщегоНазначения.ЗначениеВСтрокуXML/ЗначениеИзСтрокиXML/ЗначениеВJSON/JSONВЗначение`.

## Сериализуемость для ПовтИсп/фона/хранилища
Возвращаемое и параметры — сериализуемые (примитивы, ссылки, строки, структуры/массивы из них). Мутабельные
коллекции из ПовтИсп отдавать через `ФиксированныеДанные(...)`. См. `references/performance.md`.
