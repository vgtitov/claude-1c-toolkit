# P4 — Правка управляемых форм: дизайн

**Цель:** программно добавлять реквизиты формы, элементы (поля/группы/таблицы),
команды на управляемую форму — с byte-perfect минимальным диффом и той же
гарантией round-trip, что и метаданные. Обезличено.

## Структура (снята с реального сервера УТ)

Формат Конфигуратора: `<Объект>/Forms/<Имя>/Ext/Form.xml`, namespace
`http://v8.1c.ru/8.3/xcf/logform` (`lf`). Корень `<Form>`:
- скалярные свойства (`WindowOpeningMode`, `AutoTitle`, …);
- `<Attributes>` — реквизиты формы: `<Attribute name="X" id="N"><Type>…</Type>
  <MainAttribute>…</MainAttribute>…</Attribute>`;
- `<ChildItems>` — элементы: `<InputField name="X" id="N"><DataPath>Рекв.Поле
  </DataPath>…<ContextMenu name=".." id="M"/><ExtendedTooltip name=".." id="K"/>
  </InputField>`, а также `<UsualGroup>`, `<Table>`, `<Button>`, …;
- `<Events>`, `<Commands>`, `<Parameters>`.

EDT-формат иной (`Forms/<Имя>/Form.form`, mdclass-подобный) — отдельная фаза,
нужен реальный `.form`-образец (в текущем контуре нет).

## Ключевое отличие от метаданных: идентификация по `id`, не по `uuid`

Элементы формы нумеруются ЦЕЛЫМ атрибутом `id`, УНИКАЛЬНЫМ в пределах формы
(реквизиты, элементы, вложенные ContextMenu/ExtendedTooltip — все из одного
пространства id). Значит:
- при добавлении элемента новый `id` = (максимальный id в форме) + 1;
- элемент с вложенными частями (InputField → ContextMenu + ExtendedTooltip)
  требует НЕСКОЛЬКИХ свежих последовательных id;
- аналог `_reidentify` для форм — переприсвоение `id` (не uuid) всему поддереву.

## Операции (фазы)

- **P4.1 — `form add-attribute`** (реализуется первой): реквизит формы в
  `<Attributes>` (name + type + свежий id). Клон образца-реквизита формы или
  минимальный скелет. Основа для остального.
- **P4.2 — `form add-field`**: элемент `InputField` в `<ChildItems>`, привязка
  `DataPath` к реквизиту; свежие id для поля и вложенных ContextMenu/ExtendedTooltip.
- **P4.3 — `form add-command` / группа / колонка таблицы**: позже.

## Гарантии и границы (как в метаданных)

- byte-perfect минимальный дифф; round-trip verify платформой в тест-базе.
- Scope-граница: только БЕЗОПАСНОЕ добавление. Удаление/переименование элементов
  формы с починкой обработчиков и ссылок — вне scope (каскад → EDT/человек).
- Семантические поля-ссылки при клонировании чистятся/параметризуются (как для
  Column/URLTemplate): `DataPath` задаётся явно, не наследуется от образца.

## Проверка

- Unit: фикстура реальной структуры формы (обезличено), проверка уникальности всех
  `id` после добавления (golden), byte-perfect round-trip.
- Integration: загрузка формы в тест-базу платформой + round-trip diff.
