# Настройка «руками агента»: дай ИИ-агенту этот текст — и он всё поднимет сам

`bsl-ai-toolkit` рассчитан на то, что **сам AI-агент** (Claude Code, Cursor, Codex, Gemini CLI…) разворачивает
окружение по инструкции — человеку остаётся 3–4 действия, которые агент физически сделать не может (склонировать свои
1С-исходники, опубликовать тест-базу, выдать токены). Ниже — **готовый промпт**, который можно вставить агенту, и что он
сделает.

---

## 1. Промпт агенту (скопируй и вставь)

> Ты — инженер по окружению. Настрой на этой машине `bsl-ai-toolkit` для AI-ассистированной разработки 1С. Действуй
> по шагам, после каждого показывай вывод команды (не «готово» без вывода):
>
> 1. Определи ОС. Если репозитория ещё нет — склонируй `https://github.com/vgtitov/bsl-ai-toolkit` и зайди в него.
> 2. Запусти онбординг: на macOS/Linux — `bash onboard/onboard.sh`; на Windows — `powershell -File onboard/onboard.ps1`.
>    Он поставит скиллы, соберёт конфиги из `core/` (`sh build.sh` / `build.ps1`), профиль `.mcp.json`, git-хуки и сделает самотест.
> 3. Прогони health-check: `python scripts/doctor.py` — покажи, что зелёное, что красное.
> 4. Если чего-то из инструментов не хватает (BSL Language Server, платформа) — `python scripts/detect_tools.py --install`
>    (скачивает свободные jar'ы, прописывает env). Платящее/лицензируемое (саму платформу 1С) — НЕ ставь, попроси меня.
> 5. Проверь MCP: перечисли серверы из `.mcp.json`, скажи каких env-переменных не хватает (сверься с `.env.example`).
>    Секреты я введу сам — не проси и не печатай их.
> 6. Прогони тесты: `uv run --with mcp --with pytest pytest tests/ -q` — должны быть зелёные.
> 7. Сделай итог: что работает, что мне нужно доделать вручную (из списка ниже), и как проверить onec-code
>    (`find_object` по моему коду).
>
> Правила: факты о 1С проверяй по коду (MCP `onec-code`) и справке, не по памяти; секреты/пароли/ПДн в чат не выводи;
> ничего не применяй к ПРОД-базе; при сомнении — спроси меня.

---

## 2. Что агент сделает сам
- Клонирование репозитория и запуск `onboard/*` под нужную ОС.
- Сборку конфигов из `core/` (`build.sh`/`build.ps1`) → `CLAUDE.md`/`AGENTS.md`/`.mcp.json`/`.claude/`.
- Копирование скиллов в `~/.claude/skills` (для Claude) и раскладку профиля MCP.
- Установку git-хуков (`commit-msg`, `pre-commit` = детектор «запрос в цикле»).
- Скачивание **свободных** инструментов (`detect_tools.py --install`: BSL Language Server и пр.).
- Health-check (`doctor.py`) и прогон тестов.
- Диагностику «каких env не хватает» по `.env.example` (без ввода секретов).

## 3. Что должен сделать человек (агент физически не может)
1. **Склонировать свои 1С-исходники** (выгрузка конфигурации + расширения из EDT/Конфигуратора) в каталог `ONEC_SRC_DIR`
   (по умолчанию `~/erp-src`) — по ним ищет MCP `onec-code`. Без них агент «слеп» по коду проекта.
2. **Ввести секреты/токены** в `.env` (строки соединения, токены Zabbix/центрального onec-code) — только сам, в env, не в чат.
3. **Установить платформу 1С и (опц.) 1C:EDT** — лицензируемое ПО ставит человек по инструкции агента.
4. Для слоя данных `onec-data` — **опубликовать HTTP-сервис расширения `ai_debug` и OData на тест-базе** (это доступ к
   ОС веб-сервера, делает админ/разработчик; см. `docs/data-access-architecture.md`). Без публикации `onec-data` не работает,
   остальное — работает.
5. Проставить **версии своего проекта** (платформа, режим совместимости, конфигурация, БСП) в
   `core/skills/1c-dev/references/conventions-template.md` и карту контуров в `config/contours.example.md`.

## 4. Провайдер LLM (важно для РФ)
Агенту нужен доступ к модели. Из РФ прямой доступ к Anthropic/OpenAI/Google ограничен (`unsupported_region`) — подними
доступ на уровне окружения агента (прокси-обёртка API / RU-VDS + свой прокси / VPN на роутере). Toolkit провайдера не
диктует. Подробнее — `adapters/README.md`, `docs/ACCESS_SETUP.md`.

## 5. Проверка, что всё поднялось (агент делает сам)
- `python scripts/doctor.py` — пререквизиты + MCP из `.mcp.json` + доступы, всё зелёное.
- `uv run --with mcp --with pytest pytest tests/ -q` — тесты зелёные.
- Запрос к `onec-code` (`find_object` / поиск по коду) отдаёт результат из `ONEC_SRC_DIR`.
- Правка `*.bsl` → сработал хук `bsl_guard` (если есть «запрос в цикле» — предупредил).
