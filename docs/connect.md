# Подключение к инструментам 1С (Claude Code) — generic runbook

> Org-agnostic основа. Локализация делает поверх свою версию со своими данными (URL центра, имена профилей,
> список контуров) — см. как это сделано в производной (`docs/CONNECT.md` локализации). Здесь — механизм.

## Что получится
- **erp-1c** — поиск/чтение кода 1С (`find_object`/`search_1c`/`read_module`/`list_modules`). Источник — локальный (stdio) ИЛИ центральный (sse).
- **bsl-platform** / **bsl-ls** — справка по платформе и проверка синтаксиса (dev).
- Скиллы `1c-dev` / `1c-analyst`.

## local vs центр — что выбрать
**Ключевое различие:** `erp-1c` — это ЧТЕНИЕ кода (Claude *смотрит* код). Правка кода — это файлы рабочей копии,
которые Claude меняет обычными инструментами (Read/Edit/Grep) в открытом проекте, а НЕ через erp-1c.

- **Центр** (рекомендуется для команды/нескольких баз): код лежит на сервере, обновляется централизованно, локально
  держать клоны конфигураций НЕ нужно. Подключение — `set_token` + `switch_erp --mode central`.
- **Локально** (один разработчик / оффлайн): склонируй свои репозитории в `ONEC_SRC_DIR`, stdio-движок читает их.

**Что держать локально:** только тот код, который ты реально правишь как файлы (твоя рабочая копия). Всё остальное
(чужие контуры, типовая, библиотеки) дешевле читать из центра. Для правки своей копии локальный erp-1c-индекс не нужен —
Claude и так видит файлы проекта.

## Установка
1. Предусловия: `git`, `uv`, `java`, `claude`, `rg` в PATH.
2. `onboard` (`onboard/onboard.ps1` / `onboard.sh`; на Windows при `Restricted` — `onboard/onboard.cmd`): раскладывает
   скиллы, `.mcp.json`, `CLAUDE.md`, env, ставит no-coauthor git-хук, зовёт `switch_erp`.
3. **Центр** (если развёрнут): `scripts/set_token.(ps1|sh)` — спросит URL/токен, положит их в env пользователя,
   напечатает готовый блок для вставки в Claude Code. Затем `switch_erp --mode auto` сам уйдёт в central.

## ⚠ Безопасное хранение токена
- Токен центра (`ERP1C_TOKEN`) — секрет. **Никогда** не коммить, не клади в `.mcp.json`/файлы, не пиши в чат.
- Только переменная окружения пользователя: Windows `setx ERP1C_TOKEN ...` (+ `ERP1C_URL`); mac/Linux `export` в `~/.zshrc`.
  Проще — `scripts/set_token.*`, он делает это сам и не светит токен на экране.
- В `.mcp.json` попадает лишь плейсхолдер `Bearer ${ERP1C_TOKEN}` — Claude Code подставит значение из env при старте.
  Секрета в файле нет, `.mcp.json` безопасен.
- Утечка → ротация централизованно (на сервере — `ERP_BEARER_TOKEN`, см. `docs/docker.md`).

## Self-test
Перезапусти Claude Code → спроси «найди объект ...» — erp-1c должен ответить по коду с тегами слоёв.
Для центра из терминала: `curl $ERP1C_URL` → 401; с `-H "Authorization: Bearer $ERP1C_TOKEN"` → 200.
