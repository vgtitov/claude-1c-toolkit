# Пример: как AI ведёт задачу от тех-проекта до передачи кода

Иллюстративный сценарий (имена объектов/модулей — ПЛЕЙСХОЛДЕРЫ; в реальном проекте бери их из своего кода через MCP).
Префикс расширения здесь — `xx`. Контур: EDT + расширение.

**Сценарий:** На форме элемента справочника `Контрагенты` вывести новый реквизит расширения `xxНадёжный` (Булево),
на группу дополнительных реквизитов, безопасно и идемпотентно.

## Шаг 0. Понять и сверить
- Механизм программного изменения типовых форм у вас — модуль `xxСобытияФормСервер`, точка входа из
  переопределяемого модуля, диспетчер по `Форма.ИмяФормы`. (Проверить: `search "СобытияФормСервер"`.)
- Есть ли уже обработчик формы Контрагентов и хелпер безопасного добавления поля — проверить через MCP, переиспользовать.

## Шаг 1. Атомарные задачи
| ID | Исполнитель | Действие | Критерий |
|----|-------------|----------|----------|
| A1 | **[Разработчик/IDE]** | Заимствовать `Контрагенты` в расширение (если нужно) и добавить реквизит `xxНадёжный` (Булево) | реквизит в дереве, тип Булево |
| A2 | **[AI/код]** | В обработчик формы Контрагентов добавить идемпотентный вывод поля `xxНадёжный` на группу доп. реквизитов | поле выводится, без дубля |
| A3 | **[Разработчик/IDE]** | Проверка конфигурации в EDT → загрузка в ИБ → открыть форму | поле отображается; в базе без реквизита форма не падает |

## Шаг 2. Инструкции разработчику (НЕ-код, в EDT)
Создай реквизит `xxНадёжный` у справочника `Контрагенты`: тип **Булево**, синоним «Надёжный». Объект заимствовать в
расширение. Без этого реквизита код не привяжется — поэтому это делается ДО кода средствами IDE (IDE-гейт).

## Шаг 3. Код (по конвенциям проекта)
```bsl
Процедура ПриСозданииФормыСправочникаКонтрагенты(Форма, Отказ, СтандартнаяОбработка)

    ГруппаРеквизиты = Форма.Элементы.ГруппаДополнительныеРеквизиты;

    // идемпотентно: не добавлять, если поле уже есть
    Если Форма.Элементы.Найти("xxНадёжный") = Неопределено Тогда
        НовыйЭлемент = Форма.Элементы.Добавить("xxНадёжный", Тип("ПолеФормы"), ГруппаРеквизиты);
        НовыйЭлемент.Вид = ВидПоляФормы.ПолеФлажка;
        НовыйЭлемент.ПутьКДанным = "Объект.xxНадёжный";
    КонецЕсли;

КонецПроцедуры
```
Если в проекте есть готовый хелпер безопасного добавления поля (с проверкой наличия реквизита в БД) — использовать
его (reuse-first), а не писать свой `Элементы.Добавить`.

## Шаг 4. Проверка и тесты
- BSL LS: синтаксис, структура областей, неиспользуемые переменные.
- Граничные случаи: поле уже есть → не дублируется (`Найти`); имя формы другое → обработчик не вызывается.
- Тест разработчику: открыть форму Контрагентов, проверить отображение и отсутствие дубля при переоткрытии.

## Шаг 5. Передача (пакет)
1. Создать в EDT: реквизит `xxНадёжный` (Булево) у Контрагентов.
2. Код: изменён обработчик формы Контрагентов в `xxСобытияФормСервер`.
3. Проверить: проверка конфигурации EDT.
4. Тест: форма открывается, поле «Надёжный» отображается.
5. Загрузить в ИБ из EDT.

## Вывод
Цикл: вход → атомы с разделением [Разработчик/IDE]/[AI/код] → инструкции по не-коду → код на конвенциях проекта →
проверки граничных случаев → пакет передачи. Структуру создаёт человек в IDE, код пишет и проверяет AI.
