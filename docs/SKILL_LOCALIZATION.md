# Локализация скиллов: контракт `references/local/`

Generic-скиллы toolkit (1c-dev, 1c-analyst, 1c-expert, 1c-metadata, 1c-dba, 1c-admin-devops)
несут МЕТОДОЛОГИЮ без данных конкретной компании. Всё, что зависит от компании — версии,
контуры, конвенции имён, карты систем — приходит слоем ЛОКАЛИЗАЦИИ: файлами в
`references/local/` внутри установленного скилла.

## Как это работает

1. **Toolkit** ставит скилл в `~/.claude/skills/<skill>/` (onboard). Папки `references/local/`
   в toolkit-репозитории НЕТ — она появляется на машине пользователя.
2. **Локализация** (репозиторий компании, напр. `claude-1c-team`) своим onboard-ом раскладывает
   свои файлы в `references/local/` каждого скилла. Никаких правок generic-текста.
3. **Скилл** первым делом читает `references/local/` (если она есть): сначала `version-stack.md`,
   затем остальные карты по мере надобности. **При противоречии локальное побеждает generic.**

## Зарезервированные имена

| Файл | Что содержит | Обязателен |
|---|---|---|
| `version-stack.md` | версии платформы (по контурам, если различаются), режим совместимости, IDE, версии библиотек (БСП и др.), префиксы расширений | да — минимум локализации |
| `project-conventions.md` | заполненный `conventions-template.md` (имена, префиксы, правила проекта) | рекомендуется |
| `contour-map.md` | карта контуров/баз компании (какая база с чем вяжется, где искать) | по надобности |
| `knowledge-map.md` | карта базы знаний (id разделов Confluence/Wiki, что где лежит) | по надобности |
| `document-frames-*.md` | рамки документов компании (переопределяют дефолтные рамки) | по надобности |
| прочие `*.md` | любые карты/справочники компании (например карта метаданных ERP) | свободно |

## Правила

- Generic-текст скиллов не содержит констант компании: версии/префиксы упоминаются только как
  «см. `local/version-stack.md`». Нашёл константу компании в generic-тексте — это баг, выноси в контракт.
- Локализация НЕ добавляет файлы вне `references/local/` и не редактирует generic-файлы —
  иначе обновление скилла из toolkit затрёт правки.
- Обновление: toolkit-скилл перезаписывается целиком, `references/local/` сохраняется
  (обновляется отдельно, из репозитория локализации).
