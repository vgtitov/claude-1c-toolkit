# Карта контуров и баз — данные для скилла `1c-analyst` (ЛОКАЛИЗУЕМО)

Это **данные локализации, а не код**. Generic-шаблон: заполни под свою организацию по РЕАЛЬНОМУ коду через MCP
(`find_object`/`search`/`read_module` по обоим слоям контура). Скилл `1c-analyst` опирается на эту карту, когда
определяет контур/базу, что с чем связано и что затронет изменение.

Локализация = СВОЯ копия этого файла (напр. `config/contours.<org>.md`), подключаемая в downstream-репозитории
локализации. Generic-ядро и тело скилла менять не нужно — обновляются только эти данные. Машинная часть (какие
репозитории в какой scope-группе для поиска) живёт рядом в `config/layers.*.toml` — держи их согласованными.
