# Claude Code — специфика (поверх общих правил AGENTS.md)

Общие правила разработки на 1С — в `AGENTS.md` (импортирован строкой выше). Ниже — только Claude-специфичное.

## Скиллы (авто-подхват по описанию задачи)
`.claude/skills/` — `1c-dev`, `1c-analyst`, `1c-metadata`, `1c-admin-devops`, `1c-dba`, `1c-expert`. Формат SKILL.md
(переносим между агентами). Claude сам подгружает нужный по описанию задачи.

## Хуки (детерминированные проверки)
`.claude/settings.json` — `PostToolUse` на `Edit|Write|MultiEdit`: после правки `*.bsl` авто-прогон BSL Language Server
(MCP `bsl-ls` / CLI). Агент видит ошибки сразу, чинит без траты токенов на запуск. Плюс git-хуки
(`scripts/install_git_hooks.py`): commit-msg (чистые сообщения) + pre-commit (`bsl_guard.py` — детектор «запрос в цикле»).

## MCP
`.mcp.json` — `onec-code` (реальный код 1С), `bsl-ls` (диагностики BSL, в комплекте), `onec-ops` (эксплуатация
read-only), `onec-data` (данные живой ИБ read-only, требует публикации). Пути и креды — через env (`.env.example`).
На рабочей станции с 1C:EDT можно добавить опциональный MCP-плагин EDT для живого доступа к рабочему пространству
(ошибки, метаданные, формы, отладка) — установка и подключение в `docs/EDT_SETUP.md`.

## Поиск по коду
Встроенные Grep/Glob (ripgrep) — первичны для поиска по конфигурации/расширению. MCP `onec-code` — обёртка над этим же
поиском с авто-обнаружением слоёв контура (конфигурация + расширение). Искать в ОБОИХ слоях.

## Правило генерации конфигов
Не правь этот файл руками для общих правил — правь `core/AGENTS.md` и пересобери `sh build.sh`. Здесь — только Claude-специфика.
