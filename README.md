# claude-1c-toolkit — AI-first разработка 1С с Claude Code

Переиспользуемый продукт: **скиллы + правила + MCP-обвязка + методология** для AI-ассистированной разработки на
1С:Предприятие 8.3 (BSL). Org-agnostic — адаптируется под любую конфигурацию и расширение. Любая организация может
взять его как основу и **локализовать** под свои контуры/конвенции, не меняя ядро методологии.

## Идея
AI не угадывает 1С, а работает по РЕАЛЬНОМУ коду через MCP и официальную справку, по стандартам, и доводит код до
production-ready. Жёсткое разделение: **структуру (метаданные/формы/СКД) создаёт человек в IDE; код пишет и
проверяет AI.** Артефакты (ЧТЗ → тех-проект) превращаются в упорядоченные атомарные задачи и проверенный код.


## Ключевые принципы (что внутри методологии)
- **Спроси MCP, не угадывай** — сигнатуры/поведение только по коду и справке; после правки — BSL Language Server.
- **IDE-гейт** — структуру меняет человек в IDE (EDT/Конфигуратор), AI — код.
- **Рабочий цикл реализации (6 шагов):** понять и сверить → атомарные задачи [Разработчик/IDE]|[AI/код] →
  инструкции по не-коду → код (reuse-first БСП, производительно, по стандартам) → проверки/тесты → передача пакетом.
- **ЧТЗ = поведение, не реализация**; тех-проект = реализация; контракты до кода.
- **Reuse-first + производительность** — БСП вместо велосипедов, ПовтИсп/кэш, производительные запросы, измерять узкое место.

## Установка
```powershell
git clone <repo> && cd claude-1c-toolkit
.\onboard\onboard.ps1            # Windows
```
```bash
git clone <repo> && cd claude-1c-toolkit && bash onboard/onboard.sh   # macOS/Linux
```
onboard копирует скиллы в `~/.claude/skills`, раскладывает профиль `.mcp.json` и `CLAUDE.md`, задаёт env. Дальше
перезапусти Claude Code и подтверди MCP-серверы.

## Адаптация под свою организацию
1. В `CLAUDE.md` — проставь свои версии (платформа, режим совместимости, конфигурация, библиотеки).
2. В `skills/1c-dev/references/conventions-template.md` — заполни слои, префиксы, точки расширения, регистры,
   слепые зоны СВОЕГО проекта (по реальному коду через MCP).
3. В `mcp/dev.mcp.json` — пути к своим инструментам (через env).
Ядро методологии и стандарты остаются как есть.
