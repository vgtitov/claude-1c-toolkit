# onec_metadata — задачи на дальнейшие улучшения

Приоритезированный бэклог. Инвариант для КАЖДОЙ задачи (не повторяется в пунктах):
TDD (сначала падающий тест), byte-perfect минимальный дифф, оба формата
(Конфигуратор + EDT) где применимо, обезличенные фикстуры из реальной структуры,
применение только в ТЕСТ-базу + round-trip, adversarial-ревью перед merge.

## P1 — Редактирование существующего (главный пробел)

Сейчас операции почти все на ДОБАВЛЕНИЕ; правка существующего — только текст
запроса СКД и значение права. Это закрывает основной разрыв.

- [x] **`set-property`** ✅ — изменить скалярное свойство существующего объекта или
  реквизита по адресу (объект → [реквизит] → свойство → значение): синоним,
  комментарий, флаг (`Indexing`/`Индексирование`, `FullTextSearch`, `PasswordMode`,
  `Use`/`Использование` и т.п.). Приёмка: значение меняется, соседние узлы не
  тронуты, идемпотентно, оба формата.
- [x] **`edit-type`** ✅ — сменить/расширить тип существующего реквизита: длина/
  точность строки/числа (квалификаторы), составной тип (несколько `Type`),
  ссылка. Приёмка: корректная перестройка блока `Type` + квалификаторов.
- [x] **`edit-type` — обобщение узлов + тип с нуля** ✅ 2026-07-15 — адресация по имени
  `Attribute/Dimension/Resource/AddressingAttribute` и объект-уровень (`--member` опущен:
  `SessionParameter/Constant/DefinedType`); сборка блока `Type`/`type` и квалификатора
  (`--length` строка, `--precision`/`--scale` число) С НУЛЯ на бестиповом поле; оба формата,
  byte-perfect. Тесты: register (изм./рес.), sessionparameter/constant, untyped.
- [x] **`edit-type` — составной тип** ✅ 2026-07-15 — несколько типов в поле: `type_ref` списком
  (CLI: повторить `--type`); N `v8:Type`/`types` byte-perfect (общий `_ensure_types`), квалификаторы
  к составному запрещены, из составного обратно в одиночный сворачивается. Оба формата. Тесты `test_composite_*`.
- [ ] **`rename` ИДЕНТИФИКАТОРА (имя объекта/реквизита/процедуры)** — РИСКОВАННО: ломает ссылки,
  одинаковые имена бывают у РАЗНЫХ объектов с разным назначением. НЕ переименовывать «в лоб» всё
  найденное. Только с анализом внешних ссылок и КОНТРОЛЕМ ЧЕЛОВЕКА (каскад с починкой ссылок и
  preview→confirm уже делает `edt-mcp`; здесь — если и добавлять, то с той же гарантией).
- [ ] **Правка ПРЕЗЕНТАЦИИ (безопасно, ссылок не ломает)** — Синоним/«Наименование», Комментарий,
  заголовки: это НЕ идентификаторы. Comment уже покрыт `set-property` (скаляр); Синоним/заголовок —
  структурные узлы, отдельная операция `set-synonym`.

## P3 — Создание объектов с нуля

- [x] **Создание реквизита без образца — через ДОНОР** ✅ (`add_child --donor`):
  образец берётся из однотипного объекта конфигурации + регенерация id (версия-
  корректно). Донор — корневой уровень, элементы с порождаемым типом (ТЧ) не
  поддержаны. Пустой ChildObjects (совсем нет детей) — ещё отдельная операция.

- [ ] **HTTP-метод как вид `add_child`** (дитя URLTemplate; в EDT — вложенный
  `<methods>`, подтверждено выгрузкой ERP; в Конфигураторе — `Method`).
